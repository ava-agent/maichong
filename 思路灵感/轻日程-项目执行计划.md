## 下一步行动计划

### 第一阶段：方向验证与原型设计 (预计1周)
- [ ] **重新定义产品**：完成 `轻日程-创业方案.md` 文档的最终定稿。
- [ ] **绘制核心界面草图 (高优先级)**：用纸笔或Figma等工具，画出“可视化时间线”的核心界面和交互流程。
- [ ] **快速用户验证**：寻找3-5位目标用户（情侣、朋友），展示草图，收集关于“简洁性”和“协作意愿”的反馈。
- [ ] **技术栈初始化**：搭建Flutter开发环境，注册并初始化Supabase项目，获取AI API密钥。

### 第二阶段：MVP核心功能开发 (预计3周)
- [ ] **Week 1: 搭建“时间线”骨架**
    -   目标：完成单机版的、可视化的日程时间线。实现本地日程的增删改查。
- [ ] **Week 2: 实现“实时协作”魔法**
    -   目标：打通Supabase后端，实现用户认证、数据上云，并通过链接邀请，让多用户能实时共同编辑同一个时间线。
- [ ] **Week 3: 集成“AI助理”雏形**
    -   目标：实现基于文本的自然语言录入。用户输入一句话，能通过调用AI API，自动创建结构化的日程事件。

### 第三阶段：上线与营销启动 (第5周开始)
- [ ] **产品打磨**：根据内部测试反馈，优化UI和交互体验。
- [ ] **应用商店准备**：准备App Store和各大安卓市场的上架资料。
- [ ] **营销内容准备**：撰写第一批小红书、抖音的推广文案和视频脚本。
- [ ] **正式上线**：提交应用商店审核，并开始第一轮营销推广。

## 监控与风险
- **进度监控**：每周五进行周复盘，检查任务完成情况。
- **用户反馈**：上线后建立用户反馈群，持续收集意见。
- **风险提示**：开发过程中若遇到难题，优先保障“时间线”和“实时协作”两大核心功能的实现，其他可适当简化。
