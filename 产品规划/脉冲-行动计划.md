# 脉冲 (Mài Chōng) - 行动计划 (Todolist)

**目标**: 基于《脉冲-产品计划书》，将产品从概念落地为可供上线的MVP版本。

---

### **阶段 0: 项目奠基 (本周内完成)**
- [ ] **文档归档**: 归档所有旧的思考和命名文档，以《脉冲-产品计划书》为唯一参考基准。
- [ ] **项目创建**: 创建名为“Maichong”或“Pulse”的代码仓库。
- [ ] **环境搭建**:
    - [ ] 搭建Flutter开发环境。
    *   [ ] 注册并创建Supabase项目，获取数据库URL和API密钥。
    *   [ ] 申请并获取DeepSeek或其他LLM的API密钥。

---

### **阶段 1: MVP核心体验开发 (3周)**

#### **Week 1: “单机脉冲” - 可视化时间线**
- [ ] **UI/UX设计**: 细化时间线和“脉冲卡片”的UI设计稿。
- [ ] **前端开发**: 实现核心的时间线滚动视图。
- [ ] **本地功能**: 实现无需联网的、在本地创建、编辑、删除“脉冲事件”的功能。

#### **Week 2: “协作脉冲” - 实时同步**
- [ ] **后端集成**: 集成Supabase SDK，完成用户注册和登录功能。
- [ ] **数据上云**: 将“脉冲事件”的数据模型对接到Supabase数据库。
- [ ] **核心协作**: 实现多人实时同步编辑同一个时间线的功能。
- [ ] **邀请机制**: 开发通过链接分享、邀请新成员加入共享时间线的功能。

#### **Week 3: “智能脉冲” - AI助手 V1**
- [ ] **UI开发**: 开发AI助手的对话气泡式UI界面。
- [ ] **API集成**: 集成LLM API，并调试基础的prompt。
- [ ] **核心智能**: 实现将一段自然语言文字解析为结构化日程事件的核心功能。

---

### **阶段 2: 上线前冲刺 (1周)**
- [ ] **分享功能**: 开发“生成分享图”的功能，并设计至少1-2款分享模板。
- [ ] **Alpha测试**: 邀请5-10位最符合“组织者”画像的朋友，进行小范围内部测试，收集反馈。
- [ ] **Bug修复与优化**: 根据测试反馈，集中修复致命bug，优化核心交互体验。
- [ ] **文案打磨**: 最终确认App内的所有引导文案和提示信息。
