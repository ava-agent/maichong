## 技术栈确认

### 前端
- **框架**: Flutter 3.24+
- **状态管理**: Riverpod 3.0+
- **路由**: go_router 13.0+
- **代码生成**: freezed, json_serializable, riverpod_generator
- **本地存储**: Hive 2.2+

### 后端
- **BaaS**: Supabase (PostgreSQL + Auth + Realtime + Storage)
- **LLM**: DeepSeek API (首选) / OpenAI API (备选)

### 开发工具
- **IDE**: VS Code + Flutter扩展
- **CLI**: Supabase CLI, Flutter SDK
- **测试**: Flutter Test (内置)

## 数据库架构确认

### 核心表结构

1. **users** - 用户表
2. **timelines** - 时间线表
3. **timeline_members** - 时间线成员表
4. **events** - 事件表
5. **event_attendees** - 事件参与者表
6. **invitations** - 邀请表

### 安全策略
- Row Level Security (RLS) 已设计
- 访问策略已定义
- 权限模型: owner/admin/member
