## Flutter开发环境

### 1. 安装Flutter SDK

#### Windows

```powershell
# 1. 下载Flutter SDK
# 访问 https://docs.flutter.dev/get-started/install/windows
# 下载最新的稳定版 zip 文件

# 2. 解压到指定目录 (例如: C:\src\flutter)
# 使用PowerShell解压
Expand-Archive flutter_windows_3.24.0-stable.zip -DestinationPath C:\src\

# 3. 添加到环境变量
# 系统属性 -> 环境变量 -> Path
# 添加: C:\src\flutter\bin

# 4. 验证安装
flutter doctor
```

#### macOS

```bash
# 1. 使用Homebrew安装
brew install --cask flutter

# 或手动下载安装
# https://docs.flutter.dev/get-started/install/macos

# 2. 验证安装
flutter doctor
```

#### Linux

```bash
# 1. 下载并解压
cd ~
wget https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_3.24.0-stable.tar.xz
tar xf flutter_linux_3.24.0-stable.tar.xz

# 2. 添加到PATH
export PATH="$PATH:$HOME/flutter/bin"

# 3. 添加到 ~/.bashrc 永久生效
echo 'export PATH="$PATH:$HOME/flutter/bin"' >> ~/.bashrc
source ~/.bashrc

# 4. 验证安装
flutter doctor
```

## Supabase配置

### 1. 创建Supabase项目

```bash
# 访问 https://supabase.com
# 点击 "New Project"

# 填写项目信息:
# - Name: maichong-dev (开发环境)
# - Database Password: (生成强密码并保存)
# - Region: 选择离用户最近的区域
# - Pricing Plan: Free tier (开发阶段)

# 等待项目创建完成 (约2分钟)
```

### 2. 安装Supabase CLI

#### Windows

```powershell
# 使用npm安装
npm install -g supabase

# 或使用scoop
scoop bucket add supabase https://github.com/supabase/supabase-scoop
scoop install supabase
```

#### macOS/Linux

```bash
# 使用npm
npm install -g supabase

# 或使用Homebrew (macOS)
brew install supabase/tap/supabase

# 验证安装
supabase --version
```

### 3. 链接本地项目

```bash
# 在项目根目录执行
cd /d/project/maichong

# 登录Supabase
supabase login

# 链接项目
supabase link --project-ref YOUR_PROJECT_REF

# 获取项目信息
# Project Ref: 在项目设置中找到
```

### 4. 获取API密钥

```bash
# 从Supabase Dashboard获取:
# Settings -> API -> Project URL & anon key

# 保存以下信息:
SUPABASE_URL=https://xxxxx.supabase.co
SUPABASE_ANON_KEY=eyJhbxxxx... (anon public key)
```

### 5. 执行数据库迁移

```bash
# 创建迁移文件
supabase migration new init_schema

# 编辑迁移文件: supabase/migrations/xxxx_init_schema.sql
# 内容见 docs/数据库设计.md

# 应用迁移到本地数据库
supabase db reset

# 推送到远程数据库
supabase db push
```

## AI服务配置

### 1. DeepSeek API (推荐)

```bash
# 1. 注册DeepSeek账号
# 访问: https://platform.deepseek.com

# 2. 创建API Key
# Dashboard -> API Keys -> Create new key

# 3. 保存API密钥
DEEPSEEK_API_KEY=sk-xxxxx...

# 4. 验证API
curl https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY"
```

### 2. OpenAI API (备选)

```bash
# 1. 创建OpenAI账号
# 访问: https://platform.openai.com

# 2. 生成API Key
# Settings -> API Keys -> Create new secret key

# 3. 保存API密钥
OPENAI_API_KEY=sk-proj-xxxxx...

# 4. 充值额度 (最低$5)
```

### 3. 配置AI服务选择

```dart
// lib/core/config/ai_config.dart
enum AIProvider {
  deepSeek,
  openai,
}

class AIConfig {
  static AIProvider currentProvider = AIProvider.deepSeek;

  static String get apiKey {
    switch (currentProvider) {
      case AIProvider.deepSeek:
        return const String.fromEnvironment('DEEPSEEK_API_KEY');
      case AIProvider.openai:
        return const String.fromEnvironment('OPENAI_API_KEY');
    }
  }

  static String get baseUrl {
    switch (currentProvider) {
      case AIProvider.deepSeek:
        return 'https://api.deepseek.com/v1';
      case AIProvider.openai:
        return 'https://api.openai.com/v1';
    }
  }

  static String get model {
    switch (currentProvider) {
      case AIProvider.deepSeek:
        return 'deepseek-chat';
      case AIProvider.openai:
        return 'gpt-4o-mini';
    }
  }
}
```

## 故障排查

### Flutter相关

| 问题 | 解决方案 |
|------|----------|
| `flutter command not found` | 检查PATH环境变量是否包含flutter/bin |
| `No devices found` | 启用开发者模式 (Windows) 或安装模拟器 |
| 依赖下载失败 | 使用 `flutter pub get --verbose` 查看详细错误 |
| Hot Reload不工作 | 尝试 `flutter clean && flutter pub get` |

### Supabase相关

| 问题 | 解决方案 |
|------|----------|
| 连接超时 | 检查API URL和密钥是否正确 |
| RLS权限错误 | 确认已执行数据库迁移和RLS策略 |
| Realtime不工作 | 检查表的Realtime是否已启用 (Dashboard -> Database -> Replication) |

### AI API相关

| 问题 | 解决方案 |
|------|----------|
| 401认证失败 | 检查API密钥是否有效 |
| 429限流 | 降低请求频率或升级API套餐 |
| 超时 | 增加客户端timeout配置 |
