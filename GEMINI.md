# GEMINI.md: Project Maichong Context

This file provides instructional context for the AI assistant about the "Maichong" project.

## Project Overview

This is a **non-code project** directory dedicated to the planning, design, and strategy for a new software product named **脉冲 (Mài Chōng)**.

"脉冲 (Mài Chōng)" is conceived as an **AI-native life rhythm coordination assistant**. Its core purpose is to help modern individuals and intimate groups (like families, couples, and close friends) coordinate their schedules and activities in a seamless, visually appealing, and intelligent way. The central idea is to move beyond traditional calendars by using an AI assistant and a collaborative, timeline-based interface to make planning intuitive and fun.

## Directory Structure and Purpose

The project is organized into the following directories, each serving a specific role in the product's development lifecycle:

*   `.`: The root directory.
*   `产品规划/` (Product Planning): Contains the core strategic documents that define the product.
    *   `脉冲-产品计划书.md`: **This is the most important file in the project.** It details the product vision, target users, core features (MVP and future versions), and competitive strategy.
    *   `脉冲-行动计划.md`: Likely outlines the steps and timeline for bringing the product to life.
*   `市场调研/` (Market Research): Intended for storing analysis of competitor products, market trends, and user studies.
*   `思路灵感/` (Ideas and Inspiration): A space for brainstorming, collecting ideas, and storing inspirational materials that don't yet fit into the formal product plan.
*   `参考图片/` (Reference Pictures): For storing UI/UX design inspiration, mood boards, and other visual assets that help define the product's aesthetic.

## How to Use This Context

When working in this directory, assume the role of a product manager, designer, or strategist. The primary goal is to develop the "Maichong" product concept further.

*   **Primary Source of Truth:** Always refer to `产品规划/脉冲-产品计划书.md` for the canonical product definition.
*   **Focus on:** Product features, user experience (UX), user interface (UI) design, marketing strategy, and business modeling.
*   **Avoid:** Generating code, unless specifically asked to create a prototype based on the product plan.
